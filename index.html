Why it exists

Vibe coding needs a state model, not just another fetch wrapper.

What usually goes wrong

An agent builds a UI that reads from one server snapshot, writes with ad hoc requests, and stores local component state that drifts away from the real source of truth. The app may work on the first load, then becomes fragile after retry, offline edits, or stale responses.

What VibeLayer changes

It keeps local storage authoritative for the UI, makes writes explicit and durable, and pushes backend specifics into a transport adapter. That is the part agents can reliably compose, inspect, and repair.

Agent friendly

Why agents can use this cleanly.

Single source of truth

UI reads from `client.store`, so the agent does not maintain a second synchronized array.

Named writes

Every mutation has a name, description, and declared effects. That makes business intent explicit.

Durable drafts

`durableDraft` and conflict policies tell the agent which fields must survive offline edits.

Inspectable contract

`createAgentContract()` turns schema and mutations into machine-readable JSON for review and CI.

Backend isolation

The transport owns routes, payloads, auth, and retries, so the agent can keep UI logic clean.

Recovery first

Queue, retry, and restart recovery are built in, which matters when the app is created by codegen.
